# C4 Level 1 — System Context

Диаграмма контекста показывает маркетплейс как единую систему и её связи с пользователями и внешними системами.

```mermaid
C4Context
    title System Context — Маркетплейс

    Person(buyer, "Покупатель", "Просматривает ленту, оформляет заказы")
    Person(seller, "Продавец", "Управляет каталогом товаров и заказами")
    Person(admin, "Администратор", "Модерирует контент, управляет комиссиями")

    System(marketplace, "Маркетплейс", "Платформа для продажи товаров: каталог, заказы, платежи, персонализация")

    System_Ext(payment_provider, "Платёжный провайдер", "Stripe / ЮKassa — эквайринг")
    System_Ext(email_provider, "Email-провайдер", "SendGrid — транзакционные письма")
    System_Ext(analytics, "Аналитика", "Сбор метрик и A/B-тесты персонализации")

    Rel(buyer, marketplace, "Просматривает ленту, оформляет заказы", "HTTPS")
    Rel(seller, marketplace, "Управляет товарами и заказами", "HTTPS")
    Rel(admin, marketplace, "Администрирует платформу", "HTTPS")

    Rel(marketplace, payment_provider, "Списание средств", "HTTPS/API")
    Rel(marketplace, email_provider, "Отправка уведомлений", "HTTPS/API")
    Rel(marketplace, analytics, "События поведения", "HTTPS")
```

## Акторы

- **Покупатель** — регистрируется, получает персонализированную ленту, добавляет товары в корзину и оплачивает заказы.
- **Продавец** — создаёт и редактирует карточки товаров, отслеживает статусы заказов по своим товарам.
- **Администратор** — модерирует каталог, настраивает комиссию маркетплейса, управляет пользователями.

## Внешние системы

- **Платёжный провайдер** — проведение платежей; маркетплейс не хранит PAN/CVV, только токены и статусы.
- **Email-провайдер** — доставка уведомлений о статусах заказов.
- **Аналитика** — опциональный контур для экспериментов с алгоритмами персонализации.
